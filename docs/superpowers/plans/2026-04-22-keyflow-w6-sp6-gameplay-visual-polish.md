# KeyFlow W6 SP6 — Gameplay Visual Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three bundled gameplay-scene visual upgrades — full-width tiles (LaneAreaWidth 4f→9f + note color/scale), live combo HUD with hide-at-zero, and a user-supplied blue-gradient background image on a dedicated `ScreenSpaceOverlay` Canvas — without touching any runtime gameplay logic, without regressing SP3's GC=0 baseline, and without breaking the SP4 multi-pitch wiring.

**Architecture:** Tile width is a SceneBuilder const flowed through the already-parameterized `laneAreaWidth` SerializeFields on `TapInputHandler`/`NoteSpawner`; no runtime code change. The new `ComboHUD` MonoBehaviour polls `JudgmentSystem.Score.Combo` (not a raw `ScoreManager` — `ScoreManager` is plain C# and can't be serialized directly) and toggles `Text.enabled` so the Update loop keeps running when the HUD is hidden. The background is a UI `Image` stretch-anchored to a dedicated `BackgroundCanvas` at `sortingOrder = -100`, avoiding SpriteRenderer scaling math. A sibling `BackgroundImporterPostprocessor` (modeled on the existing `PianoSampleImportPostprocessor`) enforces `Sprite + no-mipmap + ASTC 4x4` import settings so fresh clones and re-imports stay correct.

**Tech Stack:** Unity 6000.3.13f1 Editor tooling, `AssetPostprocessor`/`TextureImporter`, `UnityEngine.UI.Canvas`/`Image`/`Text`, NUnit EditMode tests, existing `SceneBuilder.cs` + `W6SamplesWireup.cs` workflow.

## Task 1: BackgroundImporterPostprocessor + re-import + commit .meta

**Files:**
- Create: `Assets/Editor/BackgroundImporterPostprocessor.cs`
- Ensure: `Assets/Sprites/background_gameplay.png.meta` exists with Sprite/no-mipmap/ASTC settings after re-import

**Why this task exists:** The 1 MB background PNG is already committed to `Assets/Sprites/`. But without a postprocessor it imports as default Texture2D (Default) with stereo/mipmaps/RGB24 etc., which won't compress correctly for Android and won't render as a Sprite in the UI Image. A postprocessor guarantees settings survive re-imports and fresh clones, just like `PianoSampleImportPostprocessor` does for the 17 piano WAVs.

- [ ] **Step 1: Create the postprocessor**

Create `Assets/Editor/BackgroundImporterPostprocessor.cs`:

```csharp
using UnityEditor;

namespace KeyFlow.Editor
{
    // Enforces import settings for the single background_gameplay.png asset.
    // Mirrors PianoSampleImportPostprocessor so settings stick across
    // re-imports and fresh worktree checkouts.
    public class BackgroundImporterPostprocessor : AssetPostprocessor
    {
        private const string TargetPath = "Assets/Sprites/background_gameplay.png";

        private void OnPreprocessTexture()
        {
            if (assetPath != TargetPath) return;

            var importer = (TextureImporter)assetImporter;
            importer.textureType = TextureImporterType.Sprite;
            importer.mipmapEnabled = false;
            importer.isReadable = false;

            var androidSettings = importer.GetPlatformTextureSettings("Android");
            androidSettings.overridden = true;
            androidSettings.format = TextureImporterFormat.ASTC_4x4;
            androidSettings.compressionQuality = 50; // Normal (0=Fast, 50=Normal, 100=Best)
            importer.SetPlatformTextureSettings(androidSettings);
        }
    }
}
```

In `Assets/Scripts/UI/ComboHUD.cs`, replace the placeholder `Update()` with:

```csharp
private void Update()
{
    if (judgmentSystem == null || comboText == null) return;
    var score = judgmentSystem.Score;
    if (score == null) return;

    int current = score.Combo;
    if (current == lastCombo) return;
    lastCombo = current;

    if (current == 0)
    {
        if (comboText.enabled) comboText.enabled = false;
    }
    else
    {
        if (!comboText.enabled) comboText.enabled = true;
        comboText.text = current.ToString();
        TextAssignmentCount++;
    }
}
```

The sibling `BuildMainCanvas`, `BuildPauseCanvas`, `BuildResultsCanvas`, and `BuildSettingsCanvas` methods all create their canvas GameObject at scene root (no parent). Follow that convention — no `Transform parent` argument needed:

```csharp
private static void BuildBackgroundCanvas(Sprite bgSprite)
{
    var canvasGO = new GameObject("BackgroundCanvas");
    var canvas = canvasGO.AddComponent<Canvas>();
    canvas.renderMode = RenderMode.ScreenSpaceOverlay;
    canvas.sortingOrder = -100; // underneath every other Canvas
    var scaler = canvasGO.AddComponent<CanvasScaler>();
    scaler.uiScaleMode = CanvasScaler.ScaleMode.ScaleWithScreenSize;
    scaler.referenceResolution = new Vector2(720, 1280);
    scaler.matchWidthOrHeight = 0.5f;
    canvasGO.AddComponent<GraphicRaycaster>(); // present for Canvas completeness; Image's raycastTarget is false below

    var imgGO = new GameObject("BackgroundImage", typeof(RectTransform));
    imgGO.transform.SetParent(canvasGO.transform, false);
    var rt = imgGO.GetComponent<RectTransform>();
    rt.anchorMin = Vector2.zero;
    rt.anchorMax = Vector2.one;
    rt.offsetMin = Vector2.zero;
    rt.offsetMax = Vector2.zero;

    var img = imgGO.AddComponent<Image>();
    img.sprite = bgSprite;
    img.preserveAspect = false; // uniform stretch, no letterboxing
    img.raycastTarget = false;  // don't absorb taps — they go to gameplay
}
```
