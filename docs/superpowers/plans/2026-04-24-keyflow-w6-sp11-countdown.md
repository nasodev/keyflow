# W6 SP11 — Game Start Countdown (3-2-1-GO!) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Insert a 3-second "3 → 2 → 1 → GO!" countdown between `GameplayController.BeginGameplay()` and `AudioSyncManager.StartSilentSong()`, so the song clock starts at the GO! moment.

**Architecture:** Update-driven state machine on a new `CountdownOverlay` MonoBehaviour that delegates per-popup animation to a new `CountdownNumberPopup` (SP10 pattern adapted), plays `calibration_click.wav` via an `IClickPlayer` adapter, and hides the pause button for the duration. Injection seam: `GameplayController` gets an `ICountdownOverlay` reference that defers `audioSync.StartSilentSong()` into the countdown's completion callback. `SceneBuilder.BuildCountdownOverlay` instantiates a dedicated world-space canvas and wires SerializeFields.

**Tech Stack:** Unity 6000.3.13f1, C# / Unity 2022 API (not TMP), UGUI Legacy Text + `LegacyRuntime.ttf` + `Outline`, NUnit [Test] attribute. No UnityTest coroutine tests (project convention uses Update-driven state machines + `TickForTest` hooks).

**Spec reference:** `docs/superpowers/specs/2026-04-24-keyflow-w6-sp11-countdown-design.md`

---

## Implementation note — state-machine vs coroutine

The spec uses "coroutine sequencer" in §3.2 / §4.1 loosely. This plan implements `CountdownOverlay` as an **Update-driven state machine** (state transitions on `Time.time` thresholds), not a Unity coroutine, because:

1. Project convention — `JudgmentTextPopup`, `LaneGlowController` use `Update` + `TickForTest(simulatedTime)` for EditMode testability.
2. No `UnityTest` coroutine tests exist in the repo today; introducing them for SP11 alone adds a new test idiom.
3. The spec's behavioral contract (3→2→1→GO!→onComplete) is expressible as state transitions — no cross-frame async work required.

`stepDurationSec`, `popupLifetimeSec`, `goHoldSec`, `goPitch` parameters and all numeric values from the spec remain identical.

```csharp
using UnityEngine;
using UnityEngine.UI;

namespace KeyFlow.UI
{
    [RequireComponent(typeof(RectTransform))]
    [RequireComponent(typeof(Text))]
    public class CountdownNumberPopup : MonoBehaviour
    {
        private const float PunchEndT = 0.18f;
        private const float FadeStartT = 0.55f;
        private const float PunchPeak = 1.5f;

        private Text text;
        private RectTransform rt;
        private float startTime;
        private float lifetime;
        private Color baseColor;

        private void Awake()
        {
            text = GetComponent<Text>();
            rt = GetComponent<RectTransform>();
        }

        public void Activate(float startTime, float lifetime, string label, Color color)
        {
            if (text == null) text = GetComponent<Text>();
            if (rt == null) rt = GetComponent<RectTransform>();
            this.startTime = startTime;
            this.lifetime = lifetime;
            this.baseColor = color;
            text.text = label;
            text.color = color;
            rt.localScale = Vector3.one * PunchPeak;
            if (!gameObject.activeSelf) gameObject.SetActive(true);
        }

#if UNITY_EDITOR || UNITY_INCLUDE_TESTS
        internal void TickForTest(float simulatedTime) { }
#endif
    }
}
```

```csharp
        private void Update()
        {
            Tick(Time.time);
        }

        private void Tick(float now)
        {
            float elapsed = now - startTime;
            if (elapsed <= 0f) return;

            float t = elapsed / lifetime;
            if (t >= 1f)
            {
                gameObject.SetActive(false);
                return;
            }

            float scale = t < PunchEndT
                ? Mathf.Lerp(PunchPeak, 1.0f, t / PunchEndT)
                : 1.0f;
            rt.localScale = new Vector3(scale, scale, 1f);

            float alpha = t < FadeStartT
                ? 1.0f
                : Mathf.Lerp(1.0f, 0.0f, (t - FadeStartT) / (1f - FadeStartT));
            var c = baseColor;
            c.a = alpha;
            text.color = c;
        }

#if UNITY_EDITOR || UNITY_INCLUDE_TESTS
        internal void TickForTest(float simulatedTime) => Tick(simulatedTime);
#endif
```

## Task 3: `ICountdownOverlay` + `CountdownOverlay` state machine (TDD)

**Goal:** State machine that runs 3 → 2 → 1 → GO! at 1-second intervals, plays click per step (pitch 1.0 for numbers, pitch 1.5 for GO!), hides pause button during, invokes `onComplete` after `goHoldSec` hold.

**Files:**
- Create: `Assets/Scripts/UI/ICountdownOverlay.cs`
- Create: `Assets/Scripts/UI/CountdownOverlay.cs`
- Create: `Assets/Tests/EditMode/CountdownOverlayTests.cs`

### State machine shape

```
states: Idle, Step3, Step2, Step1, StepGo, PostGoHold, Complete

BeginCountdown(onComplete):
  state = Step3, startTime = Time.time
  clickPlayer.Play(1.0), popup.Activate("3", white), pauseButton.SetActive(false)

Update (or Tick(simulatedTime)):
  elapsed = now - startTime
  target state = elapsed < 1.0 → Step3
               : elapsed < 2.0 → Step2
               : elapsed < 3.0 → Step1
               : elapsed < 3.0 + goHoldSec → StepGo  (actually enters at elapsed=3.0)
               : Complete
  on state change to Step2/Step1/StepGo: clickPlayer.Play(pitch), popup.Activate(label, color)
  on state change to Complete: pauseButton.SetActive(true), onComplete()
```

Replace `BeginCountdown` body in `CountdownOverlay.cs`:

```csharp
        public void BeginCountdown(Action onComplete)
        {
            // CountdownCanvas is saved SetActive(false) to avoid SP10 ScreenManager.Start
            // race (see spec §2.3 guardrail). Reactivate on demand.
            if (!gameObject.activeSelf) gameObject.SetActive(true);
            this.onComplete = onComplete;
            this.startTime = Time.time;
            this.lastStepFired = -1;
            EnterStep(0, Time.time);
            if (pauseButtonRoot != null) pauseButtonRoot.SetActive(false);
        }

        private void EnterStep(int step, float now)
        {
            if (step == lastStepFired) return;
```
