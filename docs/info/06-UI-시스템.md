## 1. 화면 전환 모델

### `ScreenManager` (`Assets/Scripts/UI/ScreenManager.cs`)

**4개 화면 + 3개 오버레이**를 관리한다. 화면은 상호 배타적, 오버레이는 화면 위에 겹친다.

```csharp
public enum AppScreen { Start, Main, Gameplay, Results }
```

```csharp
[SerializeField] private GameObject startRoot;
[SerializeField] private GameObject mainRoot;
[SerializeField] private GameObject gameplayRoot;
[SerializeField] private GameObject resultsCanvas;

[SerializeField] private OverlayBase settingsOverlay;
[SerializeField] private OverlayBase pauseOverlay;
[SerializeField] private OverlayBase calibrationOverlay;
```

### `Replace(target)`

```csharp
public void Replace(AppScreen target) {
    HideAllOverlays();
    startRoot.SetActive(target == AppScreen.Start);
    mainRoot.SetActive(target == AppScreen.Main);
    gameplayRoot.SetActive(target == AppScreen.Gameplay);
    resultsCanvas.SetActive(target == AppScreen.Results);
    Current = target;

    // 게임플레이 진입 시 프로필별 배경 적용
    if (target == AppScreen.Gameplay)
        backgroundSwitcher.Apply(SessionProfile.Current);

    OnReplaced?.Invoke(target);
}
```

전환 시 `OnReplaced` 이벤트를 발화 — `GameplayController.HandleScreenReplaced`가 이걸 듣고 `ResetAndStart`를 호출한다.

### Android 백 버튼 처리

`ScreenManager.Update`에서 키보드 ESC 키(Android 백 버튼이 매핑됨)를 감지:

```csharp
private void Update() {
    var kb = Keyboard.current;
    if (kb == null || !kb.escapeKey.wasPressedThisFrame) return;
    HandleBack();
}
```

#### `HandleBack` 우선순위

1. Settings 오버레이 떠 있으면 → 닫기
2. Calibration 떠 있으면 → 무시 (끝까지 진행해야 함)
3. Pause 떠 있으면 → 닫기
4. 그 외엔 화면별 동작:
   - Gameplay → Pause 오버레이 띄우기
   - Results → Main으로
   - Main → Start로
   - Start → 더블 백(2초 내 두 번) 시 `Application.Quit()`

### 사용 시점

다른 코드에서 화면 전환 트리거:

```csharp
ScreenManager.Instance.Replace(AppScreen.Gameplay);
ScreenManager.Instance.ShowOverlay(ScreenManager.Instance.SettingsOverlay);
```

`ScreenManager.Instance`는 `Awake`에서 자기 자신으로 설정 (싱글톤 패턴).

## 2. 오버레이 패턴

### `OverlayBase` (`Assets/Scripts/UI/OverlayBase.cs`)

3개 오버레이(Settings, Pause, Calibration)의 base class. 다음 두 트릭이 핵심:

#### 트릭 1: `OnEnable`에서 자기 비활성화

```csharp
private void OnEnable() {
    // 첫 OnEnable 때 자동으로 비활성화 → 명시적 Show() 호출 필요
    if (!explicitlyShown) gameObject.SetActive(false);
}
```

**왜**: SceneBuilder가 만든 prefab 인스턴스는 활성 상태로 씬에 들어간다. OverlayBase가 자기 자신을 OnEnable에서 비활성화하므로, 외부 코드가 `Show()`를 부를 때까지 안 보인다. EditMode 테스트에서 attachment 완료성을 보장하는 부수 효과도 있다.

#### 트릭 2: `Show()` / `Finish()` 라이프사이클

```csharp
public virtual void Show() {
    explicitlyShown = true;
    gameObject.SetActive(true);
    OnShown();  // 서브클래스 훅
}

public virtual void Finish() {
    OnFinishing();  // 서브클래스 훅
    gameObject.SetActive(false);
    explicitlyShown = false;
}

public bool IsVisible => explicitlyShown && gameObject.activeSelf;
protected virtual void OnShown() { }
protected virtual void OnFinishing() { }
```

서브클래스는 `OnShown` / `OnFinishing` 가상 메서드만 오버라이드하면 된다 — `SetActive` 직접 호출 금지.

## 3. 4개 화면 상세

### `StartScreen` (`Assets/Scripts/UI/StartScreen.cs`)

**프로필 선택 + BGM 재생**. 가장 단순한 화면.

```csharp
[SerializeField] private Button nayoonButton;
[SerializeField] private Button soyoonButton;
[SerializeField] private AudioSource bgmSource;

private void OnEnable() {
    if (bgmSource != null) bgmSource.Play();
}

private void OnDisable() {
    if (bgmSource != null) bgmSource.Stop();
}

private void Select(Profile p) {
    SessionProfile.Current = p;
    ScreenManager.Instance.Replace(AppScreen.Main);
}
```

#### BGM의 SetActive 라이프사이클 트릭 (SP12)

`startRoot.SetActive(false)` → Unity가 자식 GameObject들에 `OnDisable`을 전파 → `StartScreen.OnDisable` → `bgmSource.Stop()`.

명시적인 cross-screen 매니저가 없다 — Unity 라이프사이클로 자연스럽게 BGM이 멈춘다. **이 패턴은 의도된 것이니 매니저로 대체하지 말 것**(SP12 reports 명시).

`bgmSource` null 가드는 EditMode 테스트(SceneBuilder 미실행) 환경 대응 — `StartScreenBgmTests`가 없으면 NullReference 발생.

### `MainScreen` (`Assets/Scripts/UI/MainScreen.cs`)

**곡 카탈로그 → 카드 리스트** 표시.

```csharp
private IEnumerator Start() {
    settingsButton.onClick.AddListener(() => settingsOverlay.Show());
    yield return SongCatalog.LoadAsync();  // catalog.kfmanifest 로드
    PopulateCards();
    populated = true;
}
```

각 곡마다 `SongCardView`를 `Instantiate`하고, 썸네일은 `StreamingAssets/thumbs/<name>.png`에서 비동기 로드(Android는 UnityWebRequest, 그 외 File.ReadAllBytes).

#### 결과 화면에서 돌아왔을 때 새로고침

```csharp
private void HandleReplaced(AppScreen target) {
    if (target == AppScreen.Main && populated) Refresh();
}
```

기존 카드들을 destroy하고 다시 만든다 — 새 최고점/별점이 반영되도록.

### `GameplayController` (게임플레이 화면 로직)

**화면 자체는 root GameObject(`gameplayRoot`)이지만**, 동작 로직은 `Assets/Scripts/Gameplay/GameplayController.cs`에 있다(UI 폴더가 아니라). 자세한 흐름은 [04-아키텍처.md](./04-아키텍처.md) "채보 로드 흐름" 참고.

### `ResultsScreen` (`Assets/Scripts/UI/ResultsScreen.cs`)

**별 팝업 + 점수 카운트업 + 재도전/홈** 버튼.

#### 별 팝업 애니메이션

코루틴 `ScalePop(starTransform, delay)`이 별마다 0.18초 간격으로 발화:

```csharp
yield return new WaitForSeconds(delay);
float t = 0f;
while (t < 1f) {
    t += Time.deltaTime / 0.3f;
    float u = 1f - (1f - t) * (1f - t);  // ease-out quad
    star.localScale = Vector3.Lerp(Vector3.one * 0.5f, Vector3.one, u);
    yield return null;
}
```

#### 점수 카운트업

`Animate` 코루틴이 0 → finalScore까지 1.2초간 보간:

```csharp
while (t < duration) {
    t += Time.deltaTime;
    float u = 1f - (1f - t / duration) * (1f - t / duration);  // ease-out
    scoreText.text = ((int)(finalScore * u)).ToString("N0");
    yield return null;
}
scoreText.text = finalScore.ToString("N0");
```

`SongSession.LastScore`를 받아 표시. `ScreenManager.Replace(AppScreen.Results)` 직후 `resultsScreen.Display(score, newRecord)`로 호출됨.

## 5. 카운트다운

### `CountdownOverlay` (`Assets/Scripts/UI/CountdownOverlay.cs`)

곡 시작 전 **3-2-1-GO** 시퀀스. 각 단계에서 클릭 음 + 숫자 팝업.

#### 핵심 시퀀스

```csharp
public void BeginCountdown(Action onComplete) {
    StartCoroutine(Run(onComplete));
}

private IEnumerator Run(Action onComplete) {
    yield return new WaitForSeconds(0.0f);  ShowStep(3);  // "3"
    yield return new WaitForSeconds(1.0f);  ShowStep(2);  // "2"
    yield return new WaitForSeconds(1.0f);  ShowStep(1);  // "1"
    yield return new WaitForSeconds(1.0f);  ShowStep(0);  // "GO!"
    yield return new WaitForSeconds(0.5f);
    Hide();
    onComplete?.Invoke();
}
```

(실제 구현은 더 정교함 — `lastStepFired` 추적으로 missed-step 복구)

#### Missed-step 복구

프레임 드롭 시 한 번에 여러 step이 진행돼야 하면 모두 발화:

```csharp
int targetStep = ComputeStepFromTime(elapsedSec);
while (lastStepFired < targetStep) {
    lastStepFired++;
    FireStep(lastStepFired);  // 클릭 + 팝업
}
```

테스트는 `IClickPlayer`(`AudioSourceClickPlayer`의 인터페이스)를 통해 클릭 발화를 가로채 검증한다.

### `CountdownNumberPopup` (`Assets/Scripts/UI/CountdownNumberPopup.cs`)

숫자 1개의 펀치 스케일 + 페이드 아웃. 0 → t = 1.0까지 자기 갱신, 끝나면 자기 비활성화.

```
스케일 펀치: 1.5 → 1.0 (lifetime의 0~18%)
y-rise: 일정 거리 위로
알파 페이드: lifetime의 55% 시점부터 시작
```

`Time.time` 절대 시각을 쓰는데, 테스트에서는 모의 시각(`SetTimeForTest`)으로 바꿀 수 있다.

### `IClickPlayer` / `AudioSourceClickPlayer`

```csharp
public interface IClickPlayer {
    void Play(float pitch);
}
```

기본 구현 `AudioSourceClickPlayer`는 `AudioSource.PlayOneShot(clip)` + pitch 조정. 테스트 mock은 호출 횟수와 pitch 시퀀스를 기록.

## 6. 피드백 디스패처

### `FeedbackDispatcher` (`Assets/Scripts/Feedback/FeedbackDispatcher.cs`)

`JudgmentSystem.OnJudgmentFeedback` 이벤트의 단일 구독자. 햅틱 + 파티클 + 텍스트 팝업을 한곳에서 분기.

```csharp
private void OnEnable() {
    judgmentSystem.OnJudgmentFeedback += Handle;
}

private void Handle(Judgment j, Vector3 worldPos) {
    if (UserPrefs.HapticsEnabled && haptic != null) haptic.Fire(j);
    if (particles != null) particles.Spawn(j, worldPos);
    if (textSpawner != null) textSpawner.Spawn(j, worldPos);
}
```

3개 의존성 모두 인터페이스(`IHapticService`, `IParticleSpawner`, `IJudgmentTextSpawner`)로 받아 EditMode 테스트가 spy/mock을 주입할 수 있다.

#### 햅틱 토글 가드

`UserPrefs.HapticsEnabled`가 false이면 햅틱은 스킵되고 시각 효과만 발화 — 사용자가 Settings에서 햅틱을 끌 수 있다.

### `FeedbackPresets` (`Assets/Scripts/Feedback/FeedbackPresets.cs`)

`ScriptableObject`. 4판정별 햅틱/파티클/텍스트 색상 프리셋. `FeedbackPrefabBuilder`가 코드로 생성하여 `Assets/ScriptableObjects/FeedbackPresets.asset`에 저장.

```csharp
[Serializable] public struct HapticPreset {
    public int durationMs;
    public int amplitude;  // 0–255
}

[Serializable] public struct ParticlePreset {
    public Color tint;
    public float size;
    public int burst;
}

public HapticPreset perfect = new() { durationMs = 15, amplitude = 200 };
public HapticPreset great   = new() { durationMs = 10, amplitude = 120 };
public HapticPreset good    = new() { durationMs = 8,  amplitude = 60  };
public HapticPreset miss    = new() { durationMs = 40, amplitude = 180 };

public ParticlePreset perfect = new() { tint = white, size = 0.45f, burst = 16 };
public ParticlePreset great   = new() { tint = cyan,  size = 0.32f, burst = 10 };
public ParticlePreset good    = new() { tint = green, size = 0.22f, burst = 6 };

public Color textPerfect = gold;
public Color textGreat   = cyan;
public Color textGood    = green;
public Color textMiss    = red;
```

값을 바꾸려면 `FeedbackPresets.asset`을 인스펙터에서 직접 편집하거나, `FeedbackPrefabBuilder.cs`의 디폴트를 고치고 `KeyFlow/Build Feedback Assets` 메뉴 재실행.

## 7. 판정 텍스트 풀

### `JudgmentTextPool` (`Assets/Scripts/Feedback/JudgmentTextPool.cs`)

12 슬롯 라운드로빈. 매 판정 시 다음 슬롯의 `JudgmentTextPopup`을 활성화.

#### 슬롯 사전 생성 (`Awake`)

```csharp
for (int i = 0; i < 12; i++) {
    var go = new GameObject($"JudgmentText_{i}");
    var text = go.AddComponent<Text>();
    text.font = LegacyRuntime.ttf;  // Unity 6 내장
    text.fontSize = 64;
    text.alignment = TextAnchor.MiddleCenter;

    var outline = go.AddComponent<Outline>();
    outline.effectColor = Color.black;
    outline.effectDistance = new Vector2(2, -2);

    var popup = go.AddComponent<JudgmentTextPopup>();
    pool.Add(popup);
    go.SetActive(false);
}
```

검은 `Outline`은 모바일 화면(저해상도, 다양한 배경)에서 가독성 확보용.

#### `Spawn` (worldPos는 무시)

```csharp
public void Spawn(Judgment j, Vector3 worldPos) {
    var popup = pool[nextIndex];
    nextIndex = (nextIndex + 1) % pool.Count;

    string text = j switch {
        Judgment.Perfect => "PERFECT",
        Judgment.Great => "GREAT",
        Judgment.Good => "GOOD",
        _ => "MISS"
    };
    Color color = presets.GetTextColor(j);
    popup.Activate(text, color);  // worldPos 미전달 → 중앙 표시
}
```

**왜 중앙 표시인가**: Magic Piano 스타일 차용. 노트 위치마다 텍스트가 흩어지면 시각이 분산되고 사용자는 못 본다. 중앙 한 곳에서 깜박이는 게 더 명확.

### `JudgmentTextPopup` (`Assets/Scripts/Feedback/JudgmentTextPopup.cs`)

활성화 시 0 → t = 1까지 자기 갱신:

```
t < 0.22:  스케일 펀치 (1.4 → 1.0)
t 전체:    y-rise (시작 y → 시작 y + 30 단위)
t > 0.55:  알파 페이드 (1 → 0)
t >= 1:    자기 비활성화 (다음 Spawn 호출까지 대기)
```

`baseLocalY`를 Activate 시점에 캡처 → 풀이 재사용해도 항상 같은 위치에서 시작.

## 8. 파티클 풀

### `ParticlePool` (`Assets/Scripts/Feedback/ParticlePool.cs`)

hit 16개 + miss 4개 풀. Hit는 라운드로빈, Miss는 마찬가지.

#### Hit/Miss 분리 풀링

```csharp
[SerializeField] private GameObject hitPrefab;   // 16
[SerializeField] private GameObject missPrefab;  // 4

private ParticleSystem[] hitPool;
private ParticleSystem[] missPool;
```

#### `Spawn` 시 프리셋 적용

```csharp
public void Spawn(Judgment j, Vector3 worldPos) {
    if (j == Judgment.Miss) {
        SpawnMiss(worldPos);
    } else {
        var ps = NextHit();
        ps.transform.position = worldPos;
        var preset = presets.GetParticle(j);

        var main = ps.main;
        main.startColor = preset.tint;
        main.startSize = preset.size;

        var emission = ps.emission;
        var burst = emission.GetBurst(0);
        burst.count = preset.burst;
        emission.SetBurst(0, burst);

        ps.Play();
    }
}
```

Miss는 prefab 고정값 사용.

#### `OnEnable`에서 Clear

```csharp
private void OnEnable() {
    foreach (var ps in hitPool) ps.Clear(true);
    foreach (var ps in missPool) ps.Clear(true);
}
```

**왜**: 결과 화면 → 재도전 시 게임플레이 root가 재활성화되는데, 이전 세션의 잔여 파티클이 풀에 남아있으면 새 세션 시작 직후 그게 한꺼번에 터진다(ghost burst). `Clear(true)`로 강제 비움.

## 9. 레인 글로우

### `LaneGlowController` (`Assets/Scripts/Feedback/LaneGlowController.cs`)

활성 홀드 중인 레인의 sprite를 사인파로 펄스.

```csharp
private readonly HashSet<int> activeLanes = new();
[SerializeField] private SpriteRenderer[] glowSprites;  // 4개 (레인별)

public void On(int lane) => activeLanes.Add(lane);
public void Off(int lane) => activeLanes.Remove(lane);

private void Update() {
    if (audioSync.IsPaused) return;

    float brightness = 0.75f + 0.25f * Mathf.Sin(Time.time * 6f);
    foreach (var lane in activeLanes) {
        var c = glowSprites[lane].color;
        c.a = brightness;
        glowSprites[lane].color = c;
    }
}
```

`HoldTracker`가 홀드 시작/종료 시 `On(lane)`/`Off(lane)` 호출.

## 11. HUD + 디버그

### `ComboHUD` (`Assets/Scripts/UI/ComboHUD.cs`)

라이브 콤보 카운터.

```csharp
[SerializeField] private Text comboText;
[SerializeField] private JudgmentSystem judgmentSystem;

private void Update() {
    var score = judgmentSystem.Score;
    if (score == null || score.Combo < 2) {
        comboText.enabled = false;
        return;
    }
    comboText.enabled = true;
    comboText.text = score.Combo.ToString();
}
```

#### 왜 폴링인가 (이벤트 미사용)

`JudgmentSystem`이 콤보 갱신 이벤트를 따로 발화하지 않는다. 대신 `ScoreManager.Combo` 프로퍼티를 폴링. 이벤트 추가하지 않은 이유:
- HUD 컴포넌트 하나만 콤보를 보여줌 — 구독자 추가 필요성 없음
- 이벤트 추가 시 `JudgmentSystem.HandleTap` 분기마다 발화 코드 필요 → 변경 부담

#### 왜 `enabled = false`인가 (SetActive 미사용)

`Component.enabled`만 토글하면 `Update`는 계속 호출된다. `GameObject.SetActive(false)`면 컴포넌트 갱신 자체가 멈춰서 다음 프레임 폴링이 안 된다.

### `HUDPauseButton` (`Assets/Scripts/UI/HUDPauseButton.cs`)

```csharp
[SerializeField] private OverlayBase pauseOverlay;
[SerializeField] private Button button;

private void Awake() {
    button.onClick.AddListener(() => pauseOverlay.Show());
}
```

19줄 짜리 단순 버튼 → 오버레이 브리지.

### `LatencyMeter` (`Assets/Scripts/UI/LatencyMeter.cs`)

private void Awake() {

- 오디오 버퍼 사이즈

`StringBuilder`로 alloc-free 포맷 — 매 프레임 GC 압박 회피.
