## 1. Unity 6000.3.13f1

### 왜 Unity 6인가
- LTS 후보 버전(2024년 6월) 중 가장 최신 안정판이다.
- 2D 기능, Input System, NUnit 통합 등 리듬게임에 필요한 핵심이 모두 내장 지원된다.
- 단일 개발자가 8주 안에 Android APK를 출하해야 하는데, 자체 엔진이나 Native(Cocos2d-x 등) 옵션은 학습 곡선이 너무 가파르다.

### 어디에 박혀 있나
- 버전: `ProjectSettings/ProjectVersion.txt` (`6000.3.13f1`)
- 패키지: `Packages/manifest.json`
- 빌드 타깃 설정: `ProjectSettings/ProjectSettings.asset`

### 업그레이드 시 주의
`SceneBuilder` + Input System 와이어링이 1.19.0 API에 의존한다. 패키지 버전을 함부로 올리면 EditMode 테스트가 깨질 수 있다.

## 2. IL2CPP + arm64-v8a (Android 전용)

### 왜 IL2CPP
- Mono 백엔드 대비 시작 시간 / 런타임 성능이 더 빠르다. 리듬게임 입력 응답성에 직결된다.
- Android 출시 요건상 64비트(arm64)가 필수, IL2CPP는 64비트 빌드를 지원한다.

### 왜 arm64-v8a 단일 타깃
- 멀티 아키텍처(armv7 + arm64) APK는 크기가 두 배에 가깝다. MVP는 출시 사이즈를 작게 유지.
- `ProjectSettings.asset`의 `AndroidTargetArchitectures: 2`(arm64만), `AndroidAllowedArchitectures: -13` 설정.

### Android SDK 범위
- Min SDK 26 (Android 8.0) — `VibrationEffect` API 도입 버전. 햅틱 기능에 필수.
- Target SDK 35 (Android 15) — Play Store 정책 충족용 최신.
- 검증 기기: Galaxy S22 (Android 16, R5CT21A31QB).

## 3. C# 어셈블리 3개

코드 베이스는 어셈블리 정의(`.asmdef`)로 명확히 분리되어 있다.

| 어셈블리 | 위치 | 용도 |
|---|---|---|
| `KeyFlow.Runtime` | `Assets/KeyFlow.Runtime.asmdef` | 모든 게임플레이 코드(런타임). `KeyFlow` 네임스페이스 + 서브모듈. |
| `KeyFlow.Editor` | `Assets/Editor/KeyFlow.Editor.asmdef` | 에디터 전용 툴(메뉴 항목, 에셋 빌더, 임포트 후처리기, gradle 훅). `Editor` 플랫폼만 포함. |
| `KeyFlow.Tests.EditMode` | `Assets/Tests/EditMode/KeyFlow.Tests.EditMode.asmdef` | NUnit 테스트. `UNITY_INCLUDE_TESTS` 게이팅, NUnit 프레임워크 precompiled reference. |

런타임 코드는 `[assembly: System.Runtime.CompilerServices.InternalsVisibleTo("KeyFlow.Tests.EditMode")]`(`JudgmentSystem.cs`에 선언)로 internal 멤버를 테스트에 노출한다. 자세한 테스트 훅 패턴은 [09-테스트-가이드.md](./09-테스트-가이드.md) 참고.

## 4. 외부 패키지

`Packages/manifest.json`에서 직접 의존하는 비기본(non-built-in) 패키지:

| 패키지 | 버전 | 용도 |

기본 모듈(`com.unity.modules.*`) 38개는 audio, androidjni, particlesystem, ui 등 Unity 6의 표준 의존성이다.

### Newtonsoft.Json 선택 이유
Unity 기본 `JsonUtility`는 `Dictionary<TKey, TValue>` 직렬화를 지원하지 않는다. `ChartData`는 `Dictionary<Difficulty, ChartDifficulty>`로 난이도별 채보를 보관해야 했고, 그래서 Newtonsoft를 도입했다(`Assets/Scripts/Charts/ChartLoader.cs:5–6`).

## 5. 오디오: Salamander Grand Piano V3

### 라이선스 + 출처
- 작자: Alexander Holm
- 라이선스: [Creative Commons CC-BY 3.0](http://creativecommons.org/licenses/by/3.0/)
- 라이선스 텍스트: `Assets/StreamingAssets/licenses/salamander-piano-v3.txt`(앱 내 노출 의무)
- 사용 부분: 17개 샘플(velocity layer 10, MIDI 36–83)을 추려 사용

### 샘플 인벤토리
`Assets/Audio/piano/`에 `.wav` 17개 — 3 반음(minor third) 간격으로 균일 배치:
```
C2v10  Ds2v10  Fs2v10  A2v10
C3v10  Ds3v10  Fs3v10  A3v10
C4v10  Ds4v10  Fs4v10  A4v10
C5v10  Ds5v10  Fs5v10  A5v10
C6v10
```

`AudioSamplePool`(`Assets/Scripts/Gameplay/AudioSamplePool.cs:78–102`)이 입력 MIDI 피치에 가장 가까운 샘플을 골라 반음 단위 피치 시프트로 정확한 음을 만든다. 그래서 `pitchSamples` 배열 17개로 MIDI 36–83 전 음역을 커버한다. 자세한 알고리즘은 [05-게임플레이-시스템.md](./05-게임플레이-시스템.md).

### 임포트 설정
`PianoSampleImportPostprocessor`(`Assets/Editor/PianoSampleImportPostprocessor.cs`)가 강제:
- `forceToMono = true`
- `loadType = DecompressOnLoad` (재생 지연 최소화)
- `compressionFormat = Vorbis`, quality 0.60
- `sampleRateOverride = 48000`
- `preloadAudioData = true`

`.meta`가 재생성되는 fresh checkout 상황에서도 임포트 설정이 살아남도록 후처리기로 강제한다. **인스펙터에서 손으로 바꾸지 말 것** — 다음 임포트 때 덮어써진다.

### 보조 오디오
- `Assets/Audio/piano_c4.wav` — pitchSamples가 비었을 때의 폴백 단음 샘플.
- `Assets/Audio/calibration_click.wav` — 20ms 노이즈 버스트 클릭. `CalibrationClickBuilder`가 결정적(seeded RNG)으로 생성.
- `Assets/Audio/bgm/piano_play_start.mp3` — StartScreen BGM (SP12에서 추가).

## 6. 채보 원본: Mutopia Project

[Mutopia Project](https://www.mutopiaproject.org/)는 Public Domain 음악의 LilyPond/MIDI 자유 저장소다. KeyFlow는 다음 3곡의 MIDI를 가져와 자체 변환 파이프라인으로 처리한다:

| MIDI 파일 | 사용 곡 |
|---|---|
| `tools/midi_to_kfchart/midi_sources/ode_to_joy.mid` | 환희의 송가 |
| `tools/midi_to_kfchart/midi_sources/clair_de_lune.mid` | Clair de Lune |
| `tools/midi_to_kfchart/midi_sources/the_entertainer.mid` | The Entertainer |

곡별 출처 attribution은 `tools/midi_to_kfchart/midi_sources/SOURCES.md`에 보관.

엘리제를 위하여(Für Elise)는 MIDI에서 변환하지 않고 **손으로 작성한 `.kfchart`** — 작곡 자체는 PD라 라이선스 문제 없음.

## 7. Python 채보 파이프라인

`tools/midi_to_kfchart/` 디렉터리. MIDI → JSON 변환을 8단계로 분리한다.

### 의존성 (`requirements.txt`)
| 패키지 | 버전 | 용도 |
|---|---|---|
| `mido` | 1.3.2 | MIDI 파일 파싱 |
| `pytest` | 8.3.3 | 단위 테스트 (10개 테스트 파일) |
| `PyYAML` | 6.0.2 | `batch.yaml` 설정 파싱 |
| `Pillow` | 10.4.0 | `tools/gen_thumbs.py` 곡 썸네일 생성 |

### 왜 Unity 외부에서 처리하는가
- MIDI 파싱과 노트 분할 알고리즘은 Unity 런타임에 들어가지 않는다 — 출시 APK는 이미 변환된 `.kfchart`만 읽는다.
- Python으로 분리하면 채보 작업자(또는 LLM)가 게임 빌드 없이 빠르게 튜닝 루프를 돌 수 있다.

자세한 단계 설명은 [08-채보-파이프라인.md](./08-채보-파이프라인.md).

## 8. 햅틱: Android VibrationEffect

`HapticService`(`Assets/Scripts/Feedback/HapticService.cs`)가 4단계 판정별로 `VibrationEffect.createOneShot(durationMs, amplitude)`을 미리 캐싱하고, 매 판정 시 재호출한다. `AndroidHapticsBridge`가 `AndroidJavaObject` 래퍼.

API 26+ 기기에만 동작 → Min SDK 26 결정의 또 다른 이유.

권한은 `AndroidManifest.xml`을 직접 작성하지 않고, `AddVibratePermission : IPostGenerateGradleAndroidProject`(`Assets/Editor/AddVibratePermission.cs`)가 gradle 프로젝트 생성 시점에 `<uses-permission android:name="android.permission.VIBRATE">`을 주입한다. 이 방식의 장점은 Unity가 자동 생성하는 `<application>` 블록을 건드리지 않는다는 점.

## 9. 폰트: LegacyRuntime.ttf

UI 텍스트는 Unity 6 내장 `LegacyRuntime.ttf` 폰트를 그대로 쓴다. 별도 한국어 폰트를 추가하지 않았는데, 이는 APK 사이즈를 작게 유지하고 라이선스 추가 부담을 피하려는 결정.

`JudgmentTextPool`(`Assets/Scripts/Feedback/JudgmentTextPool.cs`)은 텍스트에 검은 `Outline` 컴포넌트를 추가해 기기 화면에서 가독성을 확보한다.

## 10. 빌드·CI 도구

### Unity 에디터 메뉴
모든 빌드/생성 작업은 `KeyFlow/` 메뉴 하위에 있다. 자세한 건 [07-에디터와-빌드.md](./07-에디터와-빌드.md).

### Batch-mode CLI
GitHub Actions / 로컬 헤드리스 빌드용. `-batchmode -nographics -projectPath . -executeMethod <Class.Method>` 패턴.

**중요한 함정**: `-runTests`는 `-quit`와 함께 쓰면 테스트 러너가 스킵된다. 다른 모든 `-executeMethod`는 `-quit` 필수.

## 11. 무엇을 쓰지 않았나

다음은 의도적으로 채택하지 않은 옵션:

- **Native Audio 플러그인 ($35)** — W1 PoC 결과 Unity AudioSource 16채널 풀로 50–80ms 레이턴시가 충분히 잡혀서 매수 보류. 만약 저사양 기기에서 레이턴시 초과 시 ~1일 내 교체 가능하도록 `AudioSamplePool` 인터페이스를 만들어둠.
- **TextMeshPro** — UGUI 기본 `Text` + `Outline` 컴포넌트로 충분.
- **Addressables** — 단일 씬, 정적 채보 4개라 불필요.
- **Firebase / Analytics** — MVP 비목표.
- **Unity Cloud Build** — 로컬 batch-mode + 수동 `adb install`로 충분한 단계.
- **다국어(i18n)** — 한국어 UI만, `UIStrings.cs`에 const string 하드코딩.
- **Play Asset Delivery / 동적 모듈** — APK 한 개 배포.
