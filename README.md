# KeyFlow

A Magic-Piano-style rhythm game for Android, built with Unity 6. Four-lane
tap + hold notes driven by Public Domain classical MIDIs, with multi-pitch
Salamander piano samples so each tap plays the chart's melody.

## Status

Early development (W6 of an 8-week solo MVP schedule). Target device:
Samsung Galaxy S22 (Android 16, arm64-v8a).

Current feature set:
- 4-lane tap + hold gameplay with 4 difficulty-tiered judgments (Perfect / Great / Good / Miss)
- Multi-pitch audio via Salamander Grand Piano V3 (17 velocity layers, MIDI 36–83)
- Per-tap procedural calibration overlay (20 ms noise-burst click) for device latency correction
- 4 songs shipped: Ode to Joy, Für Elise, Clair de Lune, The Entertainer (Easy + Normal)
- Tiered haptic + particle feedback per judgment, with Settings toggle
- Full-width tiles on a custom blue-gradient background, live combo HUD

## Build

```bash
# Prerequisite: Unity 6000.3.13f1 (matching ProjectSettings) installed via Unity Hub.
# Android Build Support (IL2CPP) required for APK builds.

# Open the project in Unity Editor, then:
# 1. Menu: KeyFlow → Build W4 Scene (regenerates Assets/Scenes/GameplayScene.unity)
# 2. Menu: KeyFlow → Build Calibration Click (first-time setup; generates Assets/Audio/calibration_click.wav)
# 3. Menu: KeyFlow → Build Feedback Assets (first-time setup; generates particle prefabs + preset SO)
# 4. Menu: KeyFlow → ApkBuilder → Build (or via batch mode — see ApkBuilder.cs)
```

Batch-mode builds:

```bash
# Scene rebuild
"<Unity.exe>" -batchmode -nographics -projectPath . \
  -executeMethod KeyFlow.Editor.SceneBuilder.Build -quit

# Release APK
"<Unity.exe>" -batchmode -nographics -projectPath . \
  -executeMethod KeyFlow.Editor.ApkBuilder.Build -quit
```

Output: `Builds/keyflow-*.apk`.

## Tests

```bash
"<Unity.exe>" -batchmode -nographics -projectPath . \
  -runTests -testPlatform EditMode \
  -testResults Builds/test-results.xml
```

(Do NOT pass `-quit` with `-runTests`; it skips the test runner.)

Current EditMode test count: 135+.

## Project structure

```
Assets/
  Scripts/         — runtime C# (KeyFlow namespace + submodules)
  Editor/          — scene-builder tooling, asset builders, import postprocessors
  Audio/
    piano/         — 17 Salamander V3 sample WAVs
    piano_c4.wav   — fallback sample
    calibration_click.wav — procedurally generated click
  Sprites/
    background_gameplay.png — self-authored gameplay bg
    white.png, star_filled/empty.png — procedural
  StreamingAssets/
    charts/        — 4 .kfchart song files
    catalog.kfmanifest
    licenses/      — CC-BY license text for Salamander
    thumbs/        — procedurally generated song thumbnails
  Tests/EditMode/  — NUnit EditMode tests
  Scenes/GameplayScene.unity — single gameplay scene (regenerated by SceneBuilder)

tools/
  midi_to_kfchart/ — Python pipeline: Mutopia MIDI → .kfchart JSON

docs/superpowers/  — design specs, implementation plans, completion reports
```
