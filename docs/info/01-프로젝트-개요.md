# 01. 프로젝트 개요

> 이 문서는 **누구나** — 비개발자, 신규 합류자, 그리고 "이게 뭐였더라" 잊은 사람을 위한 입문서다.

## 한 문장으로

KeyFlow는 **클래식 피아노곡을 4개의 레인으로 떨어지는 빛 노트로 연주하는 Android 리듬게임**이다. 화면을 두드릴 때마다 그 곡의 다음 음이 진짜 피아노 톤으로 들리도록 만들었다.

## 어떤 게임인가

스마트폰에서 [Magic Piano](https://en.wikipedia.org/wiki/Magic_Piano)나 [Piano Tiles](https://en.wikipedia.org/wiki/Piano_Tiles)를 해본 적이 있다면, 그 장르를 떠올리면 된다. 다만 KeyFlow는:

- **빔(beam) 노트** — 위에서 아래로 1.5–2초에 걸쳐 떨어지는 빛 점 또는 막대.
- **4개의 세로 레인** — 화면을 가로로 4등분, 손가락 한 개로 노트가 도착하는 레인을 탭한다.
- **2종류 노트** — 짧은 한 번 탭(TAP)과 길게 누르는 홀드(HOLD).
- **4단계 판정** — Perfect / Great / Good / Miss. 정확도에 따라 콤보·점수·별점이 결정된다.
- **곡당 별 0~3개** — 점수의 50% / 75% / 90% 구간을 넘기면 별을 얻는다.
- **2개 난이도** — Easy / Normal. (Hard, Expert는 v1.0 이후 과제)
- **레인마다 고유한 음** — 채보가 지정한 MIDI 피치를 그 곡의 톤으로 재생한다. 즉 **탭이 곧 그 곡의 멜로디**가 된다.

세로 모드(portrait), 한 손 또는 양손 가능. 단일 손가락 동시 입력만 지원하고, 멀티 터치(2~4 손가락 동시)는 비목표(MVP 범위 외)다.

## 누가 어떻게 만들고 있나

- **개발자 1명, 8주 솔로 일정**의 MVP 프로젝트.
- 개발자 백그라운드: 풀스택 웹 경력자가 처음으로 Unity / 게임 개발에 뛰어든 케이스라 PRD에 +1주 학습 버퍼를 잡고 시작했다.
- **현재 시점**: W6 SP12까지 진행. 즉 6주차 12번째 슈퍼파워(small-batch 개선) 단계까지 완료.
- 사용자 검증 목표: 이 APK를 지인 5~10명에게 30초 이상 플레이시키고 "한 번 더 하고 싶다"는 반응을 얻으면 MVP는 성공.

## 무엇을 검증하려는 프로젝트인가

MVP는 상업용 출시가 아니라 **3가지 가설을 검증하는 프로토타입**이다.

1. **기술적 가설** — Android에서 50–80ms 이내의 체감 입력→오디오 레이턴시로 리듬게임이 동작하는가.
2. **UX 가설** — 4레인 빔 노트가 실제로 "피아노 치는 느낌"을 주는가.
3. **재미 가설** — 처음 본 사람이 30초 이상 자발적으로 플레이하는가.

수익화·소셜·진행 시스템·다국어·튜토리얼·애니메이션 컷신·리더보드는 전부 **Out of Scope**다. 이게 MVP의 핵심 정의다.

## 지금 어떤 상태인가 (W6 SP12 기준)

플레이 가능한 빌드는 다음을 포함한다:

| 영역 | 상태 |
|------|------|
| 4레인 탭 + 홀드 게임플레이 | 완성 |
| 4단계 판정(Perfect/Great/Good/Miss) | 완성 |
| 다중 피치 오디오(Salamander Grand Piano V3, 17 샘플, MIDI 36–83) | 완성 |
| 캘리브레이션(20ms 노이즈 클릭 + 8회 탭 측정) | 완성 |
| 햅틱 + 파티클 + 판정 텍스트 피드백 | 완성, 설정에서 햅틱 토글 가능 |
| 카운트다운(3-2-1-GO) + 일시정지 + 결과 화면 | 완성 |
| 출시 곡 4개(Easy + Normal) | 완성 |
| StartScreen BGM + 프로필 선택(나윤/소윤) | 완성 (SP9, SP12) |
| 한국어 UI(`UIStrings.cs`) | 완성 |
| 단일 기기 검증(Galaxy S22, Android 16) | 진행 중 |
| 다른 기기 검증(중·저사양 기기) | W6 잔여 항목 |

## 출시 곡 4개

| 곡 | 작곡가 | 출처 | 특이사항 |
|---|------|------|---------|
| 환희의 송가 (Ode to Joy) | Beethoven | Mutopia PD MIDI | 입문 |
| 엘리제를 위하여 (Für Elise) | Beethoven | **손작성 채보**(MIDI 미가공) | 작곡 자체는 PD |
| Clair de Lune | Debussy | Mutopia PD MIDI | 잔잔, 긴 곡 |
| The Entertainer | Joplin | Mutopia PD MIDI | 빠른 템포 |

각 곡은 Easy / Normal 두 난이도가 있고, 채보는 `Assets/StreamingAssets/charts/*.kfchart` 파일에 JSON으로 저장된다. 자세한 채보 형식은 [08-채보-파이프라인.md](./08-채보-파이프라인.md)에서 다룬다.

## 핵심 차별점

다른 모바일 리듬게임 대비 KeyFlow가 신경 쓴 지점:

1. **자체 캘리브레이션** — Android는 기기/OS마다 오디오 레이턴시가 천차만별이라, 첫 실행 시 사용자가 8회 탭하는 짧은 측정으로 ±500ms 범위의 오프셋을 자동 계산해 저장한다. (`CalibrationCalculator`는 mean이 아니라 median + MAD를 쓰는데, 사용자 입력의 들쑥날쑥함에 강하기 때문이다.)

2. **다중 피치 피아노 음색** — 모든 탭이 같은 "C" 소리가 아니라, 채보가 지정한 MIDI 피치를 17개 Salamander 샘플에서 골라 반음 단위로 피치 시프트해서 재생한다. 그래서 곡의 멜로디가 자연스럽게 들린다.

3. **단일 화면 구조** — `GameplayScene.unity` 하나만 존재한다. 시작·메인·게임플레이·결과 화면은 모두 같은 씬 안에서 SetActive 토글로 전환한다. 씬 전환 비용 0, 그러나 모든 와이어링이 한곳에 모인다.

4. **에디터 자동화** — `KeyFlow/Build W4 Scene` 한 메뉴로 씬을 통째로 재생성한다. 손으로 Unity 인스펙터를 만지지 않는다. 자세한 건 [07-에디터와-빌드.md](./07-에디터와-빌드.md).

## 무엇이 아닌가

오해를 줄이기 위해 명시:

- **상업용 출시작이 아니다** — Play Store 등록, 결제, 광고 SDK는 비목표.
- **소셜 / 멀티 / 리더보드 없음** — 점수는 로컬 PlayerPrefs에 저장된다.
- **iOS 빌드 없음** — Android 단일 타깃.
- **튜토리얼 컷신 없음** — 한국어 UI 텍스트로만 안내.
- **다국어 없음** — 한국어 UI만 (영문 출시는 v1.0 이후).
- **2~4 손가락 동시 탭 없음** — MVP는 단일 손가락 입력만.
