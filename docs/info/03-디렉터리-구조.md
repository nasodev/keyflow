# 03. 디렉터리 구조

> 이 문서는 **신규 개발자**가 "그 파일/그 설정이 어디 있나" 빠르게 찾도록 만든 위치 가이드다. 각 폴더의 역할과 주요 파일을 1–2줄로 정리했다.

## 최상위 트리

```
keyflow/
├── Assets/                    # Unity 에셋 + 코드 (전체 소스 99%)
├── Packages/                  # Unity 패키지 매니페스트 + lock
├── ProjectSettings/           # Unity 프로젝트 설정 .asset 파일들
├── tools/                     # Unity 외부 Python 도구 (채보 변환, 썸네일)
├── docs/                      # 설계 문서 + 가이드 (이 폴더 포함)
├── README.md                  # 영문 게임 소개 + 빌드 명령
├── CLAUDE.md / CLAUDE-KO.md   # AI 에이전트용 압축 가이드 (영/한)
├── LICENSE                    # MIT + 3rd-party 라이선스
├── .gitignore                 # Unity 표준 + 프로젝트 추가분
└── .gitattributes             # LF 줄바꿈 + Unity YAML 머지 + 바이너리 처리
```

빌드 산출물(`Builds/`, `Library/`, `Temp/`, `Logs/`)은 모두 `.gitignore`에 들어가 저장소에 없다.

## Assets/

Unity 프로젝트의 메인 디렉터리. **모든 게임 코드와 리소스가 여기에 들어간다.**

```
Assets/
├── Audio/                     # 오디오 WAV 원본
├── Editor/                    # 에디터 전용 C# 툴 (KeyFlow.Editor 어셈블리)
├── Plugins/                   # 네이티브 플러그인 (현재 비어있음)
├── Prefabs/                   # 런타임 인스턴스화 prefab
├── Resources.meta             # 빈 폴더 표지 (Resources/ 폴더는 미사용)
├── Scenes/                    # GameplayScene.unity (생성물)
├── ScriptableObjects/         # FeedbackPresets.asset
├── Scripts/                   # 모든 런타임 C# (KeyFlow.Runtime 어셈블리)
├── Sprites/                   # PNG 스프라이트
├── StreamingAssets/           # 런타임에 파일로 읽는 채보 + 썸네일 + 라이선스
├── Tests/                     # NUnit 테스트 (KeyFlow.Tests.EditMode 어셈블리)
├── Textures/                  # 앱 아이콘 등 텍스처
├── InputSystem_Actions.inputactions  # Unity Input System 액션 매핑 (자동 생성)
├── KeyFlow.Runtime.asmdef     # 런타임 어셈블리 정의
└── *.meta                     # Unity가 모든 파일에 .meta를 만든다 (커밋 필수)
```

> **`.meta` 파일을 절대 무시하지 말 것.** `.gitignore`도 `!/[Aa]ssets/**/*.meta`로 명시 보호한다. .meta가 없거나 어긋나면 인스펙터 SerializeField 와이어링이 깨진다.

### Assets/Audio/

오디오 원본. `PianoSampleImportPostprocessor`가 `piano/` 하위 WAV의 임포트 설정을 자동 강제한다.

| 파일 | 역할 |
|---|---|
| `piano_c4.wav` | `AudioSamplePool`이 pitchSamples 비었을 때 쓰는 단음 폴백 |
| `calibration_click.wav` | 20ms 노이즈 버스트 클릭 (`CalibrationClickBuilder`가 결정적 생성) |
| `piano/` (17 WAV) | Salamander Grand Piano V3 샘플, MIDI 36–83 (C2v10 ~ C6v10) |
| `bgm/piano_play_start.mp3` | StartScreen 배경음악 (SP12 추가) |

### Assets/Prefabs/

런타임에 `Instantiate`로 찍어내는 prefab.

| 파일 | 역할 |
|---|---|
| `Note.prefab` | 노트 1개 (TAP/HOLD 공용, 스케일로 길이 조절) |
| `Feedback/hit.prefab` | 판정 성공 파티클 시스템 |
| `Feedback/miss.prefab` | 판정 실패 파티클 시스템 |

`hit.prefab` / `miss.prefab`은 `FeedbackPrefabBuilder`가 코드로 생성한다. 인스펙터에서 손으로 만들지 않는다.

런타임 C# 코드. 7개 서브폴더 + 56개 `.cs` 파일.

| `CalibrationController.cs` | 캘리브레이션 오버레이 UI + 클릭 재생 + 재시도 루프. `OverlayBase` 상속. |

|---|---|

#### Scripts/Gameplay/

| 파일 | 역할 |
|---|---|
| `GameplayController.cs` | 게임 시작/완료 오케스트레이션. 채보 로드 → 캘리브레이션 → 카운트다운 → 곡 시작 → 완료 시 Results로 전환. |

| `JudgmentSystem.cs` | 탭 입력 ↔ pending 노트 매칭(closest by abs delta). 자동 미스, 홀드 시작 탭 처리. `OnJudgmentFeedback` 이벤트. |
| `JudgmentEvaluator.cs` | delta(ms) → Perfect/Great/Good/Miss. Easy 75/150/225, Normal 60/120/180. |
| `NoteSpawner.cs` | `previewMs`(2000ms) 앞서 `Note.prefab`을 인스턴스화, 난이도별 미스 윈도 적용. |
| `NoteController.cs` | 노트 1개의 위치 보간(spawnY → judgmentY) + 자동 미스 + HOLD 길이 시각화. |
| `HoldTracker.cs` | 활성 홀드들의 lane press 추적 + 250ms retrigger 오디오 + `HoldStateMachine` 위임. |
| `HoldStateMachine.cs` | Spawned → Holding → Completed/Broken FSM. `Tick(songMs, pressedLanes)` 매 프레임 호출. |
| `AudioSamplePool.cs` | 16채널 라운드로빈 `AudioSource` + MIDI 피치 → 가장 가까운 샘플 + 반음 시프트. |
| `TapInputHandler.cs` | Unity Input System으로 멀티터치 → lane index 변환, `OnLaneTap` 이벤트. |
| `ScoreManager.cs` | 점수 누적(P/G/G/M 가중치) + 콤보 + 별점(50/75/90% 구간). |
| `Judgment.cs` | `Judgment` enum, `Difficulty` enum, `JudgmentResult` 구조체. |
| `LanePitches.cs` | 채보가 피치를 안 줄 때의 기본값 — 4 레인 = C3/G3/C4/G4 (5도 계단). |

### Assets/StreamingAssets/

런타임에 **파일 시스템에서 직접 읽는** 에셋. APK에 패킹되며, Android에서는 `UnityWebRequest`로 접근.

```
StreamingAssets/
├── catalog.kfmanifest         # 출시 곡 4개의 메타데이터 JSON
├── charts/                    # .kfchart JSON 채보
├── thumbs/                    # 곡 썸네일 PNG
└── licenses/                  # 외부 자산 라이선스 텍스트
```

| 파일 | 역할 |
|---|---|
| `catalog.kfmanifest` | `{songs: [{id, title, composer, thumbnail, difficulties, chartAvailable}, ...]}` |
| `charts/beethoven_ode_to_joy.kfchart` | 환희의 송가 채보 |
| `charts/beethoven_fur_elise.kfchart` | 엘리제를 위하여 (손작성) |
| `charts/debussy_clair_de_lune.kfchart` | Clair de Lune |
| `charts/joplin_the_entertainer.kfchart` | The Entertainer |
| `thumbs/*.png` (5개) | `tools/gen_thumbs.py`가 절차적 생성한 128x128 썸네일 + `locked.png` |
| `licenses/salamander-piano-v3.txt` | Alexander Holm CC-BY 3.0 라이선스 텍스트 |

### tools/gen_thumbs.py

### docs/superpowers/

스프린트 주도 개발(spec → plan → report)의 산출물. **시점 동결** — 각 SP 시점의 결정 컨텍스트를 보존한다.

| 폴더 | 파일 명명 규칙 | 예시 |
|---|---|---|
| `specs/` | `YYYY-MM-DD-keyflow-<sprint>-<topic>-design.md` | `2026-04-19-keyflow-mvp-design.md` |
| `plans/` | `YYYY-MM-DD-keyflow-<sprint>-<topic>.md` | `2026-04-23-keyflow-w6-sp7-scenebuilder-wireup-consolidation.md` |
| `reports/` | `YYYY-MM-DD-w<X>(-sp<Y>)-<topic>-completion.md` | `2026-04-24-w6-sp12-start-screen-bgm-completion.md` |

스프린트 진행: W1(PoC) → W2(4레인 판정) → W3(채보·홀드·캘리브레이션) → W4(스크린) → W5(파이프라인) → W6(다중 피치 + SP1~SP12).

특정 결정의 "왜"를 추적할 때:
1. 영향 받은 코드 영역의 SP 번호 추측 → `specs/` 동명 파일에서 설계 의도 확인
2. `plans/` 동명 파일에서 실제 작업 분할 확인
3. `reports/` 동명 파일에서 구현 결과 + 회귀/리스크 확인
